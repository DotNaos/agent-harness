# Iteration 2 Scope

## Goal

Turn the runtime foundation into a practical daily workspace.

This iteration focuses on usability and workflow surfaces built on top of the runtime model from Iteration 1.

---

## Intended Scope

### Git workflow surface
- branch view
- runtime-aware git status
- changed files and diff visibility
- basic staging and commit flow
- clearer branch/worktree management from the UI

### Developer workflow surfaces
- terminal surface per runtime
- browser surface
- active runtime view
- agent status and activity visibility
- smoother switching between projects, runtimes, and branches

### Runtime usability improvements
- clearer runtime creation flow
- better runtime inspection
- improved lifecycle controls
- better visibility into isolated runtime resources

---

## Explicit Planning Rule

This iteration is intentionally not fully expanded yet.

Only the broad scope is defined here.
Detailed feature specs should be written when Iteration 2 becomes the active iteration.

---

## Not Yet Planned in Detail

- exact git UI scope
- exact browser integration depth
- exact terminal implementation details
- exact agent activity UX
- detailed runtime monitoring behavior
