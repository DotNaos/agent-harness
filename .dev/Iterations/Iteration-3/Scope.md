# Iteration 3 Scope

## Goal

Add enforcement, code quality controls, and steering systems on top of the stable runtime and workflow foundation.

This iteration focuses on making agent work safer, more consistent, and more aligned with project rules.

---

## Intended Scope

### Enforcement layer
- project-toolkit enforcement hooks
- agent-toolkit enforcement hooks
- code quality response contract
- design-doc adherence support
- encapsulation and boundary checks

### Planning and steering
- structured planning workflow
- two-layer planning support
- directory-level steering documents
- context injection mechanisms
- manual verification markers for design-doc updates

### Fine-tuning and safety improvements
- stronger runtime guardrails
- better quality visibility
- improved developer review support
- more consistent agent output requirements

---

## Explicit Planning Rule

This iteration is intentionally only scoped at a high level.

Detailed features should be specified later, once the exact workflow pain points from Iteration 2 are known.

---

## Not Yet Planned in Detail

- exact enforcement execution model
- exact quality-check implementation strategy
- exact steering hook format
- exact design-doc update workflow
- exact relationship between policy checks and runtime approvals

---

## Dependency

Requires Iteration 1 and Iteration 2 to provide a stable runtime and a usable workflow surface first.
